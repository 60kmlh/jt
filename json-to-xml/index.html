Guide To Converting JSON To XML

1. How to use this tool to convert JSON to XML.

Step 1: Enter the JSON text, or upload your *.JSON file directly.

Step 2: In the JSON to XML conversion process, you can click on the options to drop down and you can set different options to determine how to convert JSON to XML.

Step 3: Save the converted XML.

  • (1) Click the Copy button, you can copy the converted XML text to the clipped version.

  • (2) Enter the file name and click the Save button, you can download the converted XML text.

2. What is JSON?

  • (1) The full name of JSON is JavaScript Object Notation.

  • (2) JSON is a lightweight data interchange format. It is easy to read and write, but also easy to parse and generate by machine.

  • (3) JSON is built on two structures: A collection of name/value pairs and an ordered list of values.

3. What is XML?

  • (1) The full name of XML is eXtensible Markup Language.

  • (2) XML is a markup language much like HTML, a data representation format that can describe very complex data structures and is commonly used to transfer and store data.

  • (3) XML has a fixed structure, the first line must be <?xml version="1.0"? >, with optional encoding. Immediately after that, the DTD is optional if declared with something like <!DOCTYPE note SYSTEM "book.dtd"> is the Document Definition Type (DTD: Document Type Definition). Next is the XML document content. An XML document has one and only one root element, which can contain any number of child elements, and the elements can contain attributes.

4. 10 Differences between JSON and XML

  • (1) JSON is based on JavaScript language; XML is derived from SGML.

  • (2) JSON is a way to represent objects; XML is a markup language that uses markup structures to represent data items.

  • (3) JSON is JavaScript Object Notation; XML is Extensible Markup Language.

  • (4) XML's files are relatively difficult to read and interpret; compared with XML, JSON's files are very easy to read.

  • (5) JSON does not provide any support for namespaces; XML supports namespaces.

  • (6) JSON supports arrays; XML does not support arrays.

  • (7) JSON only supports UTF-8 encoding; XML supports various encodings.

  • (8) JSON is less secure; XML is more secure than JSON.

  • (9) JSON does not use the end tag; XML has start and end tags.

  • (10) JSON does not support comments; XML supports comments.

5. When need to convert JSON to XML?

  • (1) Describe ui structure using XML, XML data will be more consistent with human descriptions of UI layouts than JSON.

  • (2) XML is not suitable for dynamic languages but very suitable for strongly typed languages, c/c /java processing XML is easier than processing JSON.

  • (3) XML for creating new Internet languages: XHTML, WSDL, WAP, RSS, RDF, SMIL and etc.
